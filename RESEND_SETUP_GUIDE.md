# 📧 دليل إعداد خدمة البريد الإلكتروني (Resend)

## 🎯 ما هو Resend؟
Resend هي خدمة إرسال بريد إلكتروني احترافية وسهلة الاستخدام، تعمل بشكل ممتاز مع Next.js.

---

## ✅ ما تم إعداده بالفعل:

1. ✅ تثبيت حزمة `resend` في المشروع
2. ✅ إنشاء كود إرسال الإيميلات في `/src/lib/email.ts`
3. ✅ إنشاء API Route لإرسال كود التحقق
4. ✅ تحديث واجهة المستخدم لتوضيح أن الكود يُرسل للإيميل
5. ✅ إضافة `RESEND_API_KEY` في ملف `.env`

---

## 🔑 الحصول على API Key مجاناً:

### الخطوات:

1. **الدخول إلى موقع Resend:**
   - افتح المتصفح واذهب إلى: [https://resend.com/](https://resend.com/)

2. **إنشاء حساب مجاني:**
   - اضغط على "Get Started" أو "Sign Up"
   - سجل حساب جديد باستخدام:
     - البريد الإلكتروني
     - أو حساب Google/GitHub

3. **تفعيل الحساب:**
   - بعد التسجيل، ستحصل على رسالة تفعيل في بريدك الإلكتروني
   - افتح الرسالة واضغط على رابط التفعيل

4. **إنشاء API Key:**
   - سجل الدخول إلى حسابك في Resend
   - من القائمة الجانبية، اذهب إلى **"API Keys"**
   - اضغط على زر **"Create API Key"**
   - أعطِ المفتاح اسماً واضحاً مثل: `Smiley Dental Production`
   - حدد الصلاحيات: اختر **"Full Access"**
   - اضغط **"Create"**

5. **نسخ الـ API Key:**
   - بعد الإنشاء، سظهر لك المفتاح بالصيغة: `re_xxxxxxxxxxxxxxxx`
   - ⚠️ **انسخ المفتاح فوراً ولن يظهر مرة أخرى!**

---

## 📝 إضافة API Key للمشروع:

1. افتح ملف `.env` في المشروع
2. ابحث عن السطر:
   ```
   RESEND_API_KEY=re_your_api_key_here
   ```
3. استبدل `re_your_api_key_here` بالمفتاح الحقيقي من Resend
4. احفظ الملف

**مثال:**
```
RESEND_API_KEY=re_abc123def456ghi789jkl012mno345pq
```

---

## 🔄 إعادة تشغيل السيرفر:

بعد إضافة الـ API Key، يجب إعادة تشغيل السيرفر:

```bash
# أوقف السيرفر الحالي (Ctrl+C)
# ثم أعد تشغيله:
bun run dev
```

## ✨ اختبار النظام:

### طريقة 1: التسجيل الحقيقي
1. اذهب إلى صفحة التسجيل: `http://localhost:3000/auth/register`
2. أدخل بياناتك الحقيقية (الاسم، البريد الإلكتروني، كلمة المرور)
3. اضغط "إرسال كود التحقق"
4. افتح بريدك الإلكتروني (Gmail, Outlook, etc.)
5. ستجد رسالة جديدة من منصة طب الأسنان تحتوي على الكود
6. أدخل الكود في الصفحة وأكمل التسجيل

### طريقة 2: وضع التطوير
في وضع التطوير، سيظهر الكود في الـ Console وستظهر رسالة تنبيه:
```
🔐 وضع التطوير

تم إرسال كود التحقق إلى: your@email.com

في وضع التطوير، الكود هو: 123456

⚠️ ملاحظة: في الإنتاج، لن يظهر الكود هنا
```

  - دعم فني مخصص

## 🔍 استكشاف الأخطاء:

### المشكلة: لا تصل الإيميلات
**الحلول:**
1. تأكد أن `RESEND_API_KEY` صحيح في ملف `.env`
2. تأكد أن البريد الإلكتروني صحيح
3. تحقق من مجلد Spam/Junk في بريدك
4. تأكد من إعادة تشغيل السيرفر بعد إضافة المفتاح

### المشكلة: خطأ "Invalid API Key"
**الحل:**
1. تأكد أن المفتاح يبدأ بـ `re_`
2. تحقق من عدم وجود مسافات إضافية
3. جرب إنشاء مفتاح جديد من Resend

### المشكلة: الوصول إلى Console فقط
**الحل:**
- هذا يحدث في وضع التطوير بدون API Key صحيح
- تأكد من إضافة الـ API Key الصحيح وإعادة تشغيل السيرفر
