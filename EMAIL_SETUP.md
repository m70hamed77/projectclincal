# إعداد نظام إرسال الإيميل - Smiley Dental Clinic

## 📧 نظرة عامة

تم تفعيل نظام إرسال الإيميل في المشروع باستخدام خدمة **Resend**. حالياً النظام يعمل في وضع التطوير ويعرض الإيميل في console فقط.

## ✅ ما تم تنفيذه

1. ✅ تثبيت حزمة `resend`
2. ✅ إنشاء ملف `src/lib/email.ts` لإعداد نظام إرسال الإيميل
3. ✅ تحديث API لإرسال كود التحقق فعلياً
4. ✅ تحديث صفحة التحقق لعرض رسائل أفضل للمستخدم
5. ✅ إضافة رسائل تنبيه توضح مكان العثور على الكود

## 🔄 الوضع الحالي

في وضع التطوير (Development Mode)، النظام:
- ✅ يعرض محتوى الإيميل بالكامل في **console**
- ✅ يعرض الكود في console للمساعدة في الاختبار
- ⚠️ **لا يرسل** الإيميل فعلياً إلى البريد الإلكتروني

## 🚀 كيفية تفعيل الإرسال الفعلي

لإرسال الإيميلات فعلياً إلى المستخدمين، اتبع الخطوات التالية:

### 1. إنشاء حساب على Resend

1. اذهب إلى: https://resend.com/
2. أنشئ حساباً مجانياً (الخطة المجانية تسمح بـ 3,000 إيميل شهرياً)
3. أكمل عملية التحقق من البريد الإلكتروني

### 2. الحصول على API Key

1. بعد تسجيل الدخول، اذهب إلى: https://resend.com/api-keys
2. اضغط على **"Create API Key"**
3. أعطِ الاسم (مثلاً: "Smiley Dental - Development")
4. اختر الصلاحيات: **"Send"**
5. انسخ مفتاح API (سيبدأ بـ `re_`)

### 3. إضافة المفتاح إلى المشروع

افتح ملف `.env` في مجلد المشروع واستبدل:

```env
RESEND_API_KEY=re_dev_placeholder
```

بمفتاح API الخاص بك:

```env
RESEND_API_KEY=re_your_actual_api_key_here
```

### 4. إعادة تشغيل السيرفر

```bash
# أوقف السيرفر الحالي (Ctrl+C)
# ثم أعد تشغيله
bun run dev
```

## 📨 قوالب الإيميل المتاحة

### 1. إرسال كود التحقق (Email Verification)

```typescript
import { sendVerificationCode } from '@/lib/email'

await sendVerificationCode(email, code)
```

- يرسل كود تحقق من 6 أرقام
- صالح لمدة 10 دقائق
- تصميم احترافي باللغة العربية
- رسائل تنبيه واضحة

### 2. إرسال إيميل ترحيبي (Welcome Email)

```typescript
import { sendWelcomeEmail } from '@/lib/email'

await sendWelcomeEmail(email, name)
```

- رسالة ترحيب جديدة
- توجيهات للمستخدم حول الخطوات التالية

### 3. إرسال إيميل مخصص

```typescript
import { sendEmail } from '@/lib/email'

await sendEmail({
  to: email,
  subject: 'الموضوع',
  html: '<p>محتوى HTML</p>',
  text: 'محتوى نصي'
})
```

## 🧪 الاختبار

### في وضع التطوير (بدون مفتاح API):

1. املأ نموذج التسجيل
2. افتح **Console** في المتصفح (F12)
3. ستجد سطر مثل:
   ```
   [Email Verification] Code for your@email.com: 123456
   ```
4. سيتم عرض محتوى الإيميل الكامل في console:
   ```
   ============================================================
   [EMAIL SENDING SIMULATION - Development Mode]
   ============================================================
   To: your@email.com
   Subject: كود التحقق من سمايلي لطب الأسنان
   HTML Content: ...
   ============================================================
   ```

### في وضع الإنتاج (مع مفتاح API):

1. سيتم إرسال الإيميل فعلياً إلى البريد الإلكتروني
2. تحقق من:
   - صندوق الوارد (Inbox)
   - البريد المزعج (Spam)
   - البريد العشوائي (Junk)

## 📊 حدود الخطة المجانية

| الميزة | الحد |
|--------|------|
| عدد الإيميلات الشهرية | 3,000 إيميل |
| يومياً | ~100 إيميل |
| التكلفة | مجاني بالكامل |
| الدومين | يمكنك إضافة دومينك الخاص |

## 🔧 استكشاف الأخطاء

### لم يصل الإيميل؟

1. **تحقق من Console**: قد يكون في وضع التطوير
2. **تأكد من مفتاح API**: يجب أن يبدأ بـ `re_`
3. **افحص مجلد Spam**: قد يكون هناك
4. **تحقق من عنوان الإيميل**: يجب أن يكون صحيحاً

### خطأ "Invalid API Key"؟

1. تأكد من نسخ مفتاح API بالكامل
2. تحقق من أن المفتاح لا يزال فعالاً
3. تأكد من إضافته في ملف `.env` وليس `.env.example`

### خطأ "Rate limit exceeded"؟

- الخطة المجانية: 3,000 إيميل شهرياً
- انتظر بداية الشهر التالي أو ترقِ الحساب

## 📝 ملاحظات مهمة

1. **لا تلتزم ملف .env**: أضف `.env` إلى `.gitignore`
2. **استخدم دومينات حقيقية في الإنتاج**: Resend يتطلب التحقق من الدومين
3. **راقب الاستهلاك**: تأكد من عدم تجاوز الحد الشهري
4. **اختبر جيداً**: قبل الإطلاق، تأكد من أن الإيميلات تصل بشكل صحيح

## 🆚 بدائل أخرى

إذا كنت تفضل استخدام خدمة أخرى، يمكنك:

### 1. SendGrid
```bash
bun add @sendgrid/mail
```

### 2. AWS SES
```bash
bun add @aws-sdk/client-ses
```

### 3. Mailgun
```bash
bun add mailgun.js
```

لكن Resend هو الأبسط والأكثر فعالية للمشاريع الصغيرة والمتوسطة.
