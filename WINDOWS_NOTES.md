# 📝 ملاحظات سريعة للعمل على Windows

## ✅ ما تم ضبطه في المشروع

### 1. قاعدة البيانات (SQLite)
- **المشكلة السابقة:** Schema كان مكتوب `postgresql`
- **الحل:** تم تغييره إلى `sqlite`
- **الوضع الحالي:** يعمل بشكل صحيح مع ملف محلي `db/custom.db`

### 2. ملف .env
- **المشكلة السابقة:** كان يحتوي على مسار مطلق `/home/z/my-project/db/custom.db`
- **الحل:** تم تغييره إلى مسار نسبي `file:./db/custom.db`
- **الفائدة:** يعمل على جميع الأنظمة (Windows, macOS, Linux) بدون تعديل

### 3. ملف package.json
- ✅ كل السكريبتات جاهزة للعمل على Windows
- ✅ `cross-env` مثبت لتشغيل الأوامر على جميع الأنظمة
- ✅ `postinstall` يقوم بتوليد Prisma Client تلقائياً

---

## 🚀 خطوات التشغيل السريع (3 خطوات)

```bash
# 1. تثبيت الاعتماديات
npm install

# 2. إعداد قاعدة البيانات
npm run db:push

# 3. تشغيل المشروع
npm run dev
```

افتح المتصفح على: http://localhost:3000

---

## ⚡ الأوامر المهمة

```bash
npm run dev              # تشغيل السيرفر
npm run db:push          # تحديث قاعدة البيانات
npm run db:studio        # فتح واجهة إدارة قاعدة البيانات
npm run lint             # فحص الكود
npm run build            # بناء المشروع للإنتاج
```

---

## ⚠️ ملاحظات مهمة

1. **ملف .env لا يُشارك على GitHub** - يحتوي على معلومات حساسة
2. **قاعدة البيانات محلية** - لا تحتاج لـ PostgreSQL أو MySQL
3. **المشروع جاهز للعمل** - كل شيء مضبط مسبقاً
4. **إذا واجهت مشكلة** - راجع ملف `WINDOWS_SETUP.md` للحلول

---

## 🔍 التحقق من صحة المشروع

قبل البدء، تأكد من:

- ✅ Node.js 20.x أو أحدث
- ✅ npm مثبت
- ✅ ملف `.env` موجود وبه `DATABASE_URL=file:./db/custom.db`
- ✅ مجلد `db/` موجود
- ✅ ملف `schema.prisma` فيه `provider = "sqlite"`

---

## 📞 في حال وجود مشكلة

1. احذف `node_modules` و `db/custom.db`
2. أعد التثبيت: `npm install`
3. أعد إنشاء قاعدة البيانات: `npm run db:push`
4. أعد التشغيل: `npm run dev`
