# ✅ قائمة التحقق قبل البدء (Checklist)

## 1️⃣ قبل البدء - التحقق من البيئة

- [ ] Node.js مثبت (الإصدار 20.x أو أحدث)
  ```bash
  node --version
  ```

- [ ] npm مثبت
  ```bash
  npm --version
  ```

- [ ] VS Code مثبت

---

## 2️⃣ إعداد المشروع

- [ ] فتح المشروع في VS Code

- [ ] فتح Terminal في VS Code (Ctrl + `)

- [ ] تثبيت الاعتماديات
  ```bash
  npm install
  ```

---

## 3️⃣ التحقق من الملفات المهمة

- [ ] ملف `.env` موجود
- [ ] ملف `.env` يحتوي على:
  ```env
  DATABASE_URL=file:./db/custom.db
  NODE_ENV=development
  ```
- [ ] ملف `prisma/schema.prisma` يحتوي على:
  ```prisma
  datasource db {
    provider = "sqlite"
    url      = env("DATABASE_URL")
  }
  ```

---

## 4️⃣ إعداد قاعدة البيانات

- [ ] تشغيل الأمر:
  ```bash
  npm run db:push
  ```

- [ ] التأكد من ظهور رسالة النجاح:
  ```
  🚀 Your database is now in sync with your Prisma schema
  ```

- [ ] التأكد من وجود ملف `db/custom.db`

---

## 5️⃣ تشغيل المشروع

- [ ] تشغيل الأمر:
  ```bash
  npm run dev
  ```

- [ ] انتظار ظهور رسالة:
  ```
  Ready in X ms
  ```

- [ ] فتح المتصفح على: `http://localhost:3000`

- [ ] التأكد من ظهور الصفحة الرئيسية

---

## 6️⃣ اختبار الوظائف (اختياري)

- [ ] التحقق من أن الموقع يعمل بشكل صحيح
- [ ] محاولة تسجيل الدخول
- [ ] فحص Console في المتصفح للتأكد من عدم وجود أخطاء

---

## 7️⃣ أوامر مفيدة لاحقاً

- [ ] فحص الكود:
  ```bash
  npm run lint
  ```

- [ ] فتح واجهة قاعدة البيانات:
  ```bash
  npm run db:studio
  ```

- [ ] بناء المشروع للإنتاج:
  ```bash
  npm run build
  ```

---

## 🎯 المشاكل والحلول السريعة

### المشكلة: npm install فشل
**الحل:**
```bash
rm -rf node_modules
npm install
```

### المشكلة: قاعدة البيانات لا تعمل
**الحل:**
```bash
rm db/custom.db
npm run db:push
```

### المشكلة: Port 3000 مشغول
**الحل:** أغلق البرامج الأخرى أو استخدم منفذ آخر
